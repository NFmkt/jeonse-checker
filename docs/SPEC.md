# 전세대출 자격진단 서비스 — MVP 스펙 (grill-me 확정)

> 확정일: 2026-07-10 / 상태: **1단계 스펙 확정 완료** → 다음: 2단계 디자인(`@ui-ux-pro-max`)
> 배경 문서: [HANDOFF.md](../HANDOFF.md), [eligibility-criteria.md](eligibility-criteria.md)

---

## 1. 서비스 개요

무주택 전세 임차인을 대상으로, 주택도시기금 전세자금대출 상품 중 자격 요건을 충족하는
상품을 스텝형 문답으로 진단해주는 클라이언트 전용 웹 서비스. 타겟은 특정 연령대로
좁히지 않고 "무주택 전세 임차인 전체". 톤앤매너는 쉽지만 과도하게 캐주얼하지 않게
(정보 신뢰도가 중요한 도메인).

## 2. 대상 상품 범위 — 7종 전부 포함

| 구분 | 상품 |
|---|---|
| 코어 | 버팀목(일반), 청년전용 버팀목, 신혼부부전용, 신생아특례 |
| 니치(자기신고) | 전세피해 임차인 버팀목, 갱신만료 임차인 지원 버팀목, 주거취약계층 이주지원 버팀목 |

니치 3종은 조건 판정이 행정적으로 애매하므로(피해 인정, 지침 대상자 여부 등)
**사용자 자기신고(self-report) 체크박스** 방식으로 처리. 각 항목에는 "확인서/지침 대상
여부는 관할 기관에서 최종 확인됩니다" 같은 보조 설명을 첨부.

## 3. 문답 흐름 (스텝형 마법사, 모바일 퍼스트)

한 화면에 한 질문씩. 순서:

1. 이사할 집 결정 여부
2. 지역 (수도권/비수도권)
3. 전용면적
4. 전세보증금
5. 주택소유 여부 (무주택/1주택/2주택↑/공공임대 거주)
6. 결혼·가구 (미혼/신혼/자녀 등)
7. 연소득 (부부합산)
8. 순자산
9. 보유 대출
10. **특이사항** (중기재직·혁신도시·연체이력 + **니치 3종 자기신고 체크박스** 통합)
11. 나이

## 4. 판정 로직

- **계산 위치**: 클라이언트(브라우저) 전용. 서버로 개인정보(소득/자산 등) 전송·저장 없음.
- **규칙 표현**: 상품별 자격 수치를 JSON 규칙 데이터(`data/products.json` 등)로 분리.
  각 필드에 `sourceUrl`, `verifiedAt` 포함 → 매년 갱신 시 코드 안 건드리고 데이터만 수정.
- **공유/저장 기능 없음**: URL 쿼리 등에 민감정보 노출 리스크 있어 MVP 제외. "다시 하기"만 제공.

## 5. 결과 화면

- **전체 7개 상품을 모두 표시**, 각각 "가능"/"불가" 배지 + **구체적 미충족 사유**
  (예: "소득 5,200만원으로 한도 5,000만원 초과")
- 각 상품에 **대출한도 및 금리 range(최저~최고)** 표시 — ⚠️ 별도 리서치 필요(8장 참조)
- 상품 카드 하단에 출처 URL + 확인일자 표기

## 6. 법적 고지 (3가지, savehome 무관 언급은 제외 확정)

1. **정확성 면책**: "본 진단 결과는 2026-07 기준 공식 자격요건을 바탕으로 한 참고용
   시뮬레이션이며, 실제 대출 가능 여부는 기금e든든 또는 취급은행의 심사 결과에 따라
   다를 수 있습니다."
2. **개인정보 비저장 고지**: "입력하신 정보는 서버로 전송·저장되지 않으며, 브라우저를
   닫거나 새로고침하면 사라집니다."
3. **출처 고지**: 각 상품 카드 하단 "출처: 주택도시기금포털 (확인일: 2026-07-09)"

## 7. 기술스택 / 배포

- **Next.js + TypeScript + Tailwind CSS**
- **배포: Vercel** (Next.js 마찰 최소, 향후 서버 로직 확장 여지 유지)
- 반응형: 모바일 퍼스트, 데스크톱은 중앙 정렬 카드로 확장
- 완료 후 루트에 `시작 {포트}.bat` 생성 (CLAUDE.md 필수 후처리, 기존 포트 중복 금지 확인 필요)

## 8. 남은 리서치 (개발 착수 전, 별도 세션)

- [ ] 상품별 **대출한도 + 금리 range**(최저~최고, 공식 소스) — 5장 결과 화면에 필요
- [ ] 중기청→청년버팀목 통합 **공식 시행일 문구** 확정 (eligibility-criteria.md 118행 참조)
- 위 항목 완료 후 4단계(TDD 구현) 착수. 2단계(디자인)·3단계(이슈분할)는 선행 가능.

## 9. 논의했으나 보류/제외한 항목

- 결과 공유 URL: 민감정보 노출 트레이드오프로 제외
- 자동 데이터 갱신(크롤러): MVP 범위 밖, 수동 갱신 전제 (JSON 구조만 갱신 용이하게 설계)
- 대출한도/금리 상세(소득구간별): 스코프 확장 리스크 커서 "range"로 단순화, 세부는 아웃링크
