## #1. E2E 트레이서 (전체 파이프라인 최소 동작)

**Type**: AFK
**Blocked by**: None - can start immediately

### What to build
핵심 문항 5개(지역, 전용면적, 전세보증금, 연소득, 순자산)만으로 구성된 최소 문답 → 클라이언트 JSON 규칙 데이터로
"버팀목전세자금(일반)" 1개 상품만 판정 → 결과 화면(충족/미충족 배지 + 구체적 미충족 사유)까지
전체 아키텍처(문답 → JSON 규칙 판정 → 결과 렌더링)가 한 번에 관통하는 최소 버전.
목적은 기능 완성이 아니라 **파이프라인 자체의 동작 검증**.

> 2026-07-11 결정: 승인 기준의 "보증금·면적 상한 기준 정확 판정"을 완전히 만족시키기 위해
> 전세보증금 질문을 추가해 4개→5개 문항으로 확장(HANDOFF.md 0-1 참조).

### Acceptance criteria
- [ ] 5개 질문에 답하면 클라이언트에서만 계산되고 서버로 전송되지 않음(네트워크 탭 확인 가능)
- [ ] 버팀목전세자금(일반) 자격 충족/미충족이 정확히 판정됨(소득 5천만 한도, 순자산 3.45억 한도, 보증금·면적 상한 기준)
- [ ] 미충족 시 구체적 사유 텍스트 표시(예: "소득 5,200만원으로 한도 5,000만원 초과")
- [ ] 자격 규칙 수치가 JSON 데이터 파일로 분리되어 있고 코드에 하드코딩되지 않음

## #2. 전체 문답 플로우 확장 (11단계, 코어 4개 상품)

**Type**: AFK
**Blocked by**: #1

### What to build
SPEC.md 3장에 정의된 11단계 전체 문답(집 결정여부·지역·면적·보증금·주택소유·결혼가구·소득·순자산·
보유대출·특이사항·나이)을 구현하고, 코어 4개 상품(버팀목 일반·청년전용·신혼부부전용·신생아특례)
전체를 판정 대상으로 확장. 스텝형 마법사 UI(한 화면에 한 질문, 이전 단계로 돌아가기 가능).

### Acceptance criteria
- [ ] 11단계 질문이 SPEC.md 순서대로 모두 구현됨
- [ ] 각 단계에서 이전 단계로 돌아가 답변을 수정할 수 있음
- [ ] 코어 4개 상품 각각의 자격 요건(eligibility-criteria.md 1~4번)이 정확히 판정됨
- [ ] 결과 화면에 4개 상품 전체가 표시되고 각각 충족/미충족 + 사유가 나옴
- [ ] "다시 하기" 버튼으로 처음부터 재입력 가능(URL 공유 기능은 없음)

## #3. 니치 3종 자기신고 + 판정 통합

**Type**: AFK
**Blocked by**: #2

### What to build

"확인서/지침 대상 여부는 관할 기관에서 최종 확인" 보조 설명 포함). 체크된 항목에 대해 해당 니치
상품(eligibility-criteria.md 6~8번)의 자격 요건도 판정 로직에 포함시켜 결과 화면에 반영.

### Acceptance criteria
- [ ] 특이사항 단계에 니치 3종 체크박스가 기존 중기재직·혁신도시 항목과 함께 노출됨
- [ ] 각 체크박스에 보조 설명 문구가 표시됨
- [ ] 체크 시 해당 니치 상품이 결과 화면 7개 상품 목록에 포함되어 판정됨
- [ ] 체크 안 하면 니치 상품은 판정 대상에서 자연스럽게 제외(미충족이 아니라 비대상으로 구분)
- [ ] 결과 화면이 총 7개 상품(코어4 + 니치3)을 모두 표시함

## #4. 대출한도·금리 range 표시

**Type**: AFK
**Blocked by**: ~~별도 리서치~~ → **완료(2026-07-10)**. 데이터는 `docs/eligibility-criteria.md`
"대출한도·금리 range" 표 참조. 이제 착수 가능(#3 완료 후 진행).

### What to build
결과 카드 각각에 대출한도(최대 금액)와 금리 range(최저~최고)를 추가 표시. 리서치가 자격요건처럼
JSON 규칙 데이터에 `sourceUrl`, `verifiedAt`과 함께 추가된다는 전제로, 기존 상품 데이터 구조를
확장해서 반영.

### Acceptance criteria
- [ ] 리서치 산출물의 대출한도·금리 range 데이터가 상품별 JSON에 반영됨(출처·확인일 포함)
- [ ] 결과 카드에 한도/금리 range가 명확한 수치로 표시됨(범위 형태, 소득구간별 세부 금리표는 다루지 않음)
- [ ] 데이터 미확보 상품은 "공식 미확인" 표기로 대체되고 추정값을 표시하지 않음
